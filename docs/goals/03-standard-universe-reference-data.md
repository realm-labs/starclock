# Goal 03 — Complete Standard Simulated Universe Reference Data

## Objective

Prepare a complete, auditable Version 4.4 Standard Simulated Universe data pack
and production-ready Excel/Sora authoring surface before universe runtime work.
The goal ends with validated data, schemas, generated readers and behavioral
review fixtures; it does not implement `starclock-mode-universe`.

The normative data boundary is
[Standard Simulated Universe reference-data contract](../23-standard-simulated-universe-reference.md).

## Terminal outcome

- machine-readable manifests account for every included main-world record;
- normalized reference rows contain exact or explicitly approximate mechanics,
  bilingual summaries and row-level provenance;
- all nine Worlds and all nine selectable Paths are accounted;
- Blessings/upgrades, Resonances/Formations, Curios/states, Occurrences/choices,
  services/currencies, Ability Tree effects and encounter pools are complete;
- `Universe.xlsx` and required Activity/Rule binding workbooks are generated by
  the approved `openpyxl` bootstrap and visually reviewed;
- Sora 0.3.0 validates and exports the workbooks reproducibly and generated Rust
  readers can load every row;
- coverage and review fixtures report 100% `DataReady` with no blocking gap;
- every batch and final clean-checkout gate is committed with evidence.

## Non-goals

- universe runtime, controllers, CLI runs or golden full runs;
- DLC-specific modes or temporary event variants;
- account rewards, achievements, assets, story/dialogue or planar item data;
- guessing hidden behavior and labeling it exact;
- JSON-direct runtime loading.

## Execution rules

- Use the pinned released source first and public pages only for cross-checks
  and gaps. Do not use leaks or preview/beta data.
- Python `openpyxl` is the workbook authoring/inspection adapter requested for
  this goal. Sora remains the schema/export authority.
- Bootstrap regenerates complete new workbooks and refuses to overwrite an
  edited authoring workbook without an explicit clean target.
- Every fact maps to evidence; long source descriptions and assets stay out of
  Git.
- Keep main-world pool membership distinct from shared-table and DLC-only rows.
- Keep one implementation batch active and commit it atomically with the ledger.
- Commit subjects use
  `<type>(<scope>): <batch-id> <imperative summary>`.
- Source files stay below 1,200 LOC unless a reviewed exception is recorded.

## Acceptance

- manifests, pack index and workbook exports regenerate byte-identically;
- all main-world content categories report 100% `DataReady`;
- every row resolves bilingual fields and exact evidence;
- authoritative decimals never round through Python/Excel floats;
- all workbook sheets pass load, formula/error, header, type, validation,
  reference and visual clipping checks;
- Sora check/build/codegen/export/load has no drift;
- no universe data enters the Goal 01 combat runtime or creates a JSON runtime
  path; the isolated Universe Sora bundle is staging/review evidence only;
- public docs distinguish prepared data from unimplemented runtime behavior;
- repository, dependency, source-size and both prior release contracts pass.
