## Decisions

| Date | Decision | Rationale |
|---|---|---|
| 2026-07-24 | Goal 04 remains a historical release and its evidence is not regenerated. | Goal 05 is an additive semantic/runtime revision, not retroactive relabeling. |
| 2026-07-24 | Do not implement general multi-pending tasks in Goal 05. | Standard Universe is sequential; this goal closes actual observed integration gaps. |
| 2026-07-24 | Reference battle projection is test-only after migration. | Production completion must prove real combat behavior. |
| 2026-07-24 | Retain documented approximations where exact public data is absent. | Integration must not fabricate authoritative values. |
| 2026-07-24 | Treat room content sources as opaque upstream identities and bind an Occurrence only through an exact variant-source match. | Numeric coincidence (`0`, `22`, `32`, `101`, and similar values) is not provenance and previously risked selecting fabricated events. |
| 2026-07-24 | Treat frozen shop pool/price-formula keys as offer-provider identities, not concrete prices. | Goal 03 did not publish generated offer rows; a trusted concrete offer must bind content, positive price and digest before the atomic purchase handler runs. |
| 2026-07-24 | Do not expose revival or random two-Blessing enhancement as an always-legal production option. | Their exact eligibility requires participant-life and inventory-cardinality predicates not yet present in Activity condition IR; the concrete handlers exist and fail atomically until the offer compiler owns those predicates. |
| 2026-07-24 | Define Curio ownership as inventory plus state plus charges, with lifecycle events as an Activity boundary ledger. | Inventory-only acquisition made charged Curios unusable and allowed contribution state to become orphaned; the complete tuple now commits or rolls back together. |
| 2026-07-24 | Materialize Ability Tree boundary values as private replacement-style Activity counters. | This gives run, battle and carry compilers one deterministic operation pipeline while keeping normalized rows and numeric implementation private. |
| 2026-07-24 | Reserve collision-checked combat ID ranges for Standard Universe rule, bundle, source and modifier bindings. | Content identities must be stable and composable without reusing game IDs or requiring a central mode branch; core-catalog collisions fail compilation. |
| 2026-07-24 | Treat a RuleBundle binding declaration as input to P3-B2, not proof of event-rule execution. | Static modifiers can already validate through combat, but event-driven Blessing/Resonance/Curio semantics require materialized Rule IR and real battle fixtures. |
| 2026-07-24 | Materialize every structured encounter immediately while separating definition match from runtime-stat accuracy. | Goal 01 bundles only 13/86 referenced definitions and no full broader scaling set; deterministic rank proxies plus the named executable-stat policy make all requests valid without falsely labeling missing mechanics or numbers exact. |
| 2026-07-24 | Separate pre-mode participant-lock identity from the mode-composed runtime combatant digest. | A Path, Blessing, Curio or Ability Tree contribution must be able to alter the battle assembly without pretending the player's locked loadout changed. |
| 2026-07-24 | Roll back the no-draw battle-start marker after nested-executor infrastructure failure instead of fabricating a Faulted result. | `BattleFault` belongs only to combat; restoring the prior Activity hash makes budget, projection and adapter failures genuinely retryable. |
| 2026-07-24 | Use a revisioned input/state/event-shape commitment in P3-B3 and reserve payload-direct event encoding for P4-B1. | Real execution needs a deterministic EventDigest now, while replay owns the complete event payload codec and must give that stronger representation a distinct compatibility revision. |
