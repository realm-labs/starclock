# Character Implementation Matrix

This table is the behavioral coverage checklist for the public roster snapshot. “Major extensions” lists the features beyond an ordinary authored Basic/Skill/Ultimate/Talent. It is not a dependency order and does not claim that exact coefficients, Traces, Eidolons, or Sora rows have been imported. Those use the terminal states in [Content data and coverage](../15-content-data-and-coverage.md).

Codes used below: **AA** action advance/extra action; **BR** Break/Toughness extension; **CP** copied/injected/forced ability; **DT** DoT; **EL** Elation subsystem; **FD** field/Zone/Territory; **HP** HP cost/loss tally/defeat interception; **LK** ally or enemy link/mark; **RS** unusual resource/team resource; **SM** summon or memosprite; **TF** transformation/ability replacement; **TR** triggered attack/additional damage; **TT** damage/heal/Toughness tally.

## Gate for implementation readiness

A row is ready to implement only when its referenced primitives have deterministic tests and its versioned data definition supplies every coefficient, cost, chance, cap, duration, hit split, and retarget policy. “Released” does not mean those values have already been imported into this repository.

The two announced rows must remain disabled in normal simulations until their definitions pass schema validation after release. Their profiles intentionally contain no guessed numbers.
